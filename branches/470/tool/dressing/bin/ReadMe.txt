・dressingとは
  ゲーム向けスクリプトエンジンです。
  言語にsquirrelを採用しています。


・dressingを起動すると自動的に boot/entrypoint.nut を実行しようとします
・boot/entrypoint.nut がない場合はファイル選択ダイアログが出ます
・core/XXX にベースとなるライブラリが入っているので見るとわかりやすいかもしれません

・動作の仕組み
  それぞれのコードを読んで欲しいので、概要を書きます
  dressing はシーン間の移動をすることで処理を進めます
  シーンとはタイトル画面だったりコンフィグ画面だったり、大きめの区切りです
  IScene を継承して実装します。
  
  シーン内ではTaskとDrawObjectという物が存在しており
  この２つのクラスを継承して、シーン内の処理を進めます
  この２つのクラスは毎フレーム(秒間６０フレーム)シーンから呼び出されるので
  そこに独立した処理を書きます
  
  シーンとシーンのつなぎは IGameRoutineクラスを継承して実装します


・Task と DrawObject の違い
  両方とも毎フレームUpdateFrame() が呼ばれるし、使い方は同じなのですが
  Task は処理をするだけ、DrawObject は画面表示も行うという違いがあります
  例えば弾作成アルゴリズムはTaskでいいですが、実際に表示させる弾はDrawObjectのほうがいいでしょう


・よくはまるワナ
  親クラスの constructor() を呼んでない
  クラスインスタンス作成時に () をつけ忘れている
